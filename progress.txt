### Build fixes (critical — production build was broken)
- [x] **Edge Runtime crypto** — `middleware.ts` used Node.js `crypto.randomBytes` (not supported in Edge Runtime). Replaced with Web Crypto API `crypto.getRandomValues` + `btoa`.
- [x] **csrf.ts Edge Runtime** — Split `generateCsrfToken` and `validateCsrfToken` into `csrf-utils.ts` (Edge-safe, no `server-only` import). `csrf.ts` kept for `requireCsrf` (server actions only). Replaced `createHash` with constant-time char comparison.
- [x] **'use server' re-exports** — `local-auth.actions.ts` re-exported from `local-password.actions.ts` and `local-user.actions.ts`. Turbopack cannot resolve these. Removed re-exports, updated all 20+ importers to import directly from source files.
- [x] **'use server' non-async export** — `local-user.actions.ts` exported `getModulesForRole` (non-async function) which violates Next.js 15 `'use server'` rules. Moved to `src/lib/get-modules-for-role.ts` (plain module). Updated all 3 importers.
- [x] **Missing i18n keys** — `login-carousel.tsx` uses `useTranslations('public.cards.carousel')` but `public.cards.carousel` didn't exist in any locale file (only `private.main.cards.carousel` existed). Added `public.cards.carousel` to all 4 locale files (en, uk, pl, de).

- [x] `twoFactor*` fields already removed from Prisma schema (audit was incorrect)
